Tournament Consensus — N biased planners, one synthesized plan
v0.1.0 · MIT
Nick Krzemienski
2026-04-22

Three architectures share the same seven invariants. They differ only in how stage 4 generates its plan. Alloy's stage 4 runs N parallel Prometheus variants under different biases and hands the result to a Synthesizer that composes — never plans. Momus audits the blend. Red Team attacks the blend. Oracle renders bird's-eye verdict. Hephaestus builds the real artifact. Atlas serializes.

01The Tournament

Stage 4 · Plan (Alloy-specific)

N Prometheus-Alloy variants fan out in parallel. Each receives the same Metis directives and probe report, but a different bias. After all N complete, the Synthesizer reads them together and composites one coherent plan — it cannot write novel phases.

On validate FAIL, Alloy routes to Intent Gate — not to Synthesizer. A failed synthesis suggests the bias mix was wrong; the whole tournament re-runs with the failure folded into new Metis directives.

03The Seven Invariants

These apply identically to Cast, Alloy, and Temper. An anneal run that violates any invariant is invalid and must not emit.

Red team always runs. Three parallel adversaries. Not a flag. Cannot be disabled.
Validation always runs. Hephaestus builds the real artifact. Empty evidence → FAIL.
Dual output. Every run produces an Opus 4.7 semantic-XML prompt AND a plan directory.
Skill enrichment automatic. Probe scans ~/.claude/skills/ and project skills.
Unbounded re-loop on FAIL. Failure folds into the next run's constraints. Not a retry.
Parallelization by default. Red team, validators, and Alloy planners fan out always.
Category routing. --type ultrabrain|deep|quick — never --model.

04Cost Profile · Alloy vs Cast vs Temper

Architecture Stage-4 shape Spawns Wall-clock Best for
Cast One planner, one shot ~8 ~4 min Bug fixes, scoped refactors, clear specs
Alloy N parallel biased planners → Synthesizer ~8 + 2N (~18 at N=5) ~6 min Novel architecture, greenfield, high-stakes
Temper One plan, iteratively deepened ~8 × depth ~7 min at depth 3 Well-scoped but complex; convergence-friendly

05Install · Usage

Install

/plugin marketplace add /Users/nick/Desktop/anneal/alloy
/plugin install anneal-alloy@anneal-alloy-dev

Usage

/anneal-alloy:anneal "<task>"                    # N=5 default
/anneal-alloy:anneal "<task>" --versions 3        # cheaper
/anneal-alloy:anneal "<task>" --versions 7        # richer
/anneal-alloy:anneal "<task>" --loop              # unbounded re-loop
/anneal-alloy:anneal "<task>" --type ultrabrain    # category routing

Refused flags

--no-red-team            # red team is invariant 1
--no-validate            # validation is invariant 2
--versions 1             # use /anneal-cast:anneal
--versions 8+            # synthesizer signal-to-noise collapses
